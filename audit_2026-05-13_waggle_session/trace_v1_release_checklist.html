Round 3 amendments — 10 more substantive issues

  • L3.2 / L5.4 scoping wrong. Must check same-instance equality (full Actor: type AND id), NOT same-type. Plus guard against single-actor sessions. Type-only check false-fires on system→system and multi-AI sessions (real production data exists in ~/.trace/sessions/).
  • L3.2 directly contradicts an existing test. tests/test_decision_guards.py:84-99 is test_human_self_resolves_clean with docstring "humans can change their mind". The test must be rewritten, not just supplemented.
  • L5.7 wrong diagnosis. scratchpad.py:182-186 already auto-derives from len(session.events) (verified by Round 3a). The "27 vs 28" off-by-one is in session.summary — human-authored prose passed to trace_end_session(summary=...). Recommendation: remove L5.7 or redirect to a summary-text/count discrepancy detector.
  • L5.6 trigger too sensitive. Round 3b ran grep on ~20 production sessions — all returned 0 tool_calls. Original threshold fires on every session, becoming noise. Revise to ≥10 contributions + client="Claude Code" AND render as advisory hint (no warning counter). Guard against environment is None.
  • L5.5 hint not warning. Rename orphan_discovery_warning_count → orphan_discovery_hint_count; render at lower severity tier. Detector is heuristic; "warning" overweights it.
  • L5.2 .strip() the snippet before allow-list lookup, so leading whitespace doesn't break absence-marker detection.
  • L6.2 underestimated. The PROV-O qualified-influence pattern is exporter architecture work (~5h, not 2h). The exporter has no qualified-influence node anywhere today; this requires new bundle sections.
  • L9.0 fixture migration missing. ~15 fixtures across 6 test files reference fields being changed. Must precede L9.1 (canonical regression). ~3-4h budget not in plan.
  • Layer 11 missing entirely. 10 adoption-surface items: README, project CLAUDE.md, CONTRIBUTING.md, installer asset (CLAUDE_BLOCK.md), decision-audit.sh hook, ADR creation, PROV namespace policy, schema $id cascade, CHANGELOG migration callouts, optional --upgrade flag for trace-mcp-init.
  • FastMCP wrapper signature. server.py's log_tool_call wrapper needs the new host and parent_event_id kwargs added too — implicit in L4.6 but worth calling out.

Recommendation: apply these amendments inline (already done below) and start implementation. Round 4 marginal value is below cost.

These were considered and excluded. Each has a rationale; do not re-add without one of those rationales being addressed.

  • Rejected: New subagent_dispatch event type (bloats every consumer; an Agent dispatch is a tool call by every operational measure).

  • Rejected: Hard-required conversation_snippet schema field (pushes controllers to fabricate; sparse honest > dense fabricated).
  • Rejected: Retroactive event injection for existing sessions (fabricates provenance).
  • Rejected: Schema version bump to 0.5.0 (all changes additive; reserve 0.5 for actually-breaking changes like the eventual mcp_servers → tools rename).
